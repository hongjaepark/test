Swagger UI (FastAPI 내장)

코드를 기반으로 자동 생성되는 인터랙티브 API 문서 및 테스트 도구

Swagger UI는 OpenAPI Specification(API 명세 표준)을 기반으로 API를 시각화하고, 브라우저에서 직접 테스트해볼 수 있는 인터랙티브 문서를 제공하는 도구입니다. FastAPI는 이를 기본적으로 내장하고 있어, 개발자가 별도의 작업 없이도 강력한 API 문서를 가질 수 있게 해줍니다.

1. ✍️ 코드 = 문서 (Code as Documentation)

FastAPI는 개발자가 작성한 파이썬 코드를 분석하여 API 문서를 자동으로 생성합니다.

  • 경로 연산(Path Operation): @app.post("/plan")과 같이 정의된 엔드포인트가 문서에 자동으로 등록됩니다.
  • Pydantic 모델: 요청 본문(Request Body)이나 응답 모델(Response Model)로 지정된 Pydantic 클래스(예: schemas.UserInput)의 구조, 타입, 제약 조건(Field의 description 등)이 문서에 상세히 반영됩니다.
  • 이를 통해 API 문서를 별도로 작성하고 관리해야 하는 번거로움이 사라지고, 코드와 문서가 항상 일치하게 됩니다.

2. 🚀 브라우저에서 직접 API 테스트

FastAPI 애플리케이션을 실행하고 /docs 경로에 접속하면, 모든 API 엔드포인트 목록을 볼 수 있습니다. 여기서 각 API를 펼쳐 'Try it out' 버튼을 누르면 직접 테스트가 가능합니다.

  • 파라미터나 요청 본문(JSON)을 입력하고 'Execute' 버튼을 누르면, 실제 서버로 요청이 전송됩니다.
  • 서버의 응답 코드(200, 404 등), 응답 본문, 헤더 등을 즉시 확인할 수 있어, 프론트엔드 개발 전 백엔드 로직을 검증하고 디버깅하는 데 매우 유용합니다.

3. 💡 future-planner 프로젝트에서의 역할

이 프로젝트에서 Swagger UI는 백엔드 개발자의 핵심 테스트 및 디버깅 도구로 활용됩니다.

  • 프론트엔드(Streamlit)와 연동하기 전에, FastAPI 서버의 /plan 엔드포인트가 예상대로 동작하는지 독립적으로 테스트할 수 있습니다.
  • 예를 들어, Swagger UI에서 직접 {"monthly_income": 3000000, "risk_appetite": "aggressive"}와 같은 JSON 데이터를 입력하여 LangGraph 워크플로우를 트리거하고, 스트리밍되는 응답 조각들을 확인할 수 있습니다.

결론적으로, Swagger UI는 복잡한 AI 에이전트 파이프라인을 API 뒤에 숨기고, 잘 정의된 인터페이스를 통해 안정적으로 호출할 수 있도록 돕는 백엔드의 '품질 보증서'이자 '테스트 장비' 역할을 수행합니다.

4. 🖥️ 실제 사용 예시

uvicorn src.app:app --reload --host 0.0.0.0 --port 8000 명령어로 서버를 실행한 후 /docs 주소에 접속하면 다음과 같은 작업을 할 수 있습니다.

  • API 엔드포인트 목록 확인: app.py에 정의된 모든 API 경로(예: / 와 /chat)와 사용 가능한 HTTP 메서드(GET, POST 등)를 한눈에 볼 수 있습니다.
  • 요청/응답 형식 파악: 각 API가 어떤 형식의 데이터를 요청(QueryRequest)하고 어떤 형식으로 응답(QueryResponse)하는지 명확하게 알 수 있습니다.
  • API 직접 테스트: 가장 유용한 기능으로, 웹 페이지에서 'Try it out' 버튼을 눌러 직접 API를 호출하고 실시간으로 응답 결과를 확인할 수 있습니다. 예를 들어, /chat 엔드포인트에 질문을 입력하고 실행하면 RAG 체인이 동작하여 답변을 반환하는 것을 바로 테스트할 수 있습니다.
