AI 어시스턴트 활용 예시: 프롬프트 요구사항

아래는 Python Streamlit과 SQLite를 사용한 간단한 CRUD 웹앱 생성을 AI에게 요청하는 프롬프트 예시입니다. 명확하고 구조화된 요구사항을 통해 원하는 결과물을 효과적으로 얻어내는 과정을 보여줍니다.

1. 기본 목표 및 기술 스택

  • 목표: 간단한 사용자 관리(CRUD) 웹 애플리케이션 제작 (testdb.py)
  • 언어: Python
  • UI 프레임워크: Streamlit
  • 데이터베이스: SQLite
  • 데이터 처리: Pandas

2. 데이터베이스 설정 (sample.db)

  • users 테이블 생성: id (PK, Auto-increment), name (Text), age (Integer)
  • CRUD 기능 함수 모듈화
  • [중요] 쿼리 시 id 값은 int()로 명시적 형 변환하여 타입 오류 방지
  • UPDATE/DELETE 실행 후 cursor.rowcount로 1개 행만 변경되었는지 검증하여 데이터 무결성 확보

3. Streamlit UI 구성

  • 사용자 추가: st.form 사용, 이름/나이 입력. 이름 누락 시 경고.
  • 사용자 목록: st.data_editor 사용, 최신순(id 내림차순) 정렬.
  • 단일 선택: '선택' 체크박스 열을 추가하여 한 번에 한 명만 선택 가능하도록 구현.
  • 읽기 전용: 데이터 열(ID, 이름, 나이)은 수정 불가 처리.
  • 검색: st.text_input으로 대소문자 무관 이름 검색 기능 구현.
  • 페이지네이션: 한 페이지에 10개씩 목록 표시, 이전/다음 버튼으로 페이지 이동.

4. 사용자 수정 및 삭제

  • 버튼 활성화: 목록에서 사용자를 선택해야 '수정', '삭제' 버튼 활성화.
  • 수정: st.dialog를 사용하여 선택된 사용자 정보가 채워진 수정 창 표시.
  • 삭제: "정말로 OOO님을 삭제하시겠습니까?"와 같은 확인 절차 후 삭제 실행.

5. 상태 관리 및 안정성

  • 상태 관리: st.session_state로 선택된 사용자 ID, 페이지 번호, 검색어 등 관리.
  • UI 갱신: 상태 변경 작업 후 st.rerun() 호출로 즉시 UI 업데이트.
  • 선택 해제: 검색 또는 페이지 이동 시 기존 선택 자동 해제.
  • 키 동적 생성: st.data_editor의 key를 동적으로 생성하여 상태 꼬임 방지.
  • 로깅: 주요 기능 실행 시 logging으로 정보 기록.
